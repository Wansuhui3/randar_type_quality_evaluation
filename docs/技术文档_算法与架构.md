# 无真值 Type 输出质量评估系统 — 技术文档

> 版本：v4.0 | 日期：2026-07-23

---

## 一、系统定位

在无外部真值（相机/人工标注）条件下，评估车载雷达目标 Type 分类输出的**自洽性与稳定性**。核心用途：发现不可靠输出、比较软件版本、筛查待标注样本。

不输出：分类准确率、Recall、F1、属性矛盾率。

---

## 二、核心算法

### 2.1 唯一判据

有效评估距离内，航迹 Type 是否发生跳变：

- `transition_count_in == 0` → **pass**（类型输出一致）
- `transition_count_in > 0` → **fail**（类型输出不一致）

零容差设计：Track_Age >= 10 已给予雷达充分稳定时间，此后任何 Type 变化均为输出质量问题。跳变到 Uncertain(7) 同样判 fail。

### 2.2 有效距离筛选

不裁剪帧，只筛选哪些跳变计入判定：

| 雷达位置 | 有效距离 | 判定条件 |
|----------|----------|----------|
| 前雷达 (flr/fr) | 100m | Dx <= 100 |
| 后雷达 (rlr/rr) | 50m | \|Dx\| <= 50 |

超出有效距离的跳变记录但不计入判定（曲线图中以橙色空心圆标注）。

设计依据：裁剪帧会产生 40.8% 伪跳变（中间帧被删导致相邻帧 Type 不连续），因此采用"不裁帧、只筛选跳变"方案。

### 2.3 跳变上下文分类 (A/B/C/D)

每次跳变根据 Measured 状态变化分类：

| 代码 | 条件 | 含义 | 严重程度 |
|------|------|------|----------|
| A | M: 1→2 | 进入预测态时跳变 | 中 |
| B | M: 2→1 | 重新捕获时跳变 | 中 |
| C | M: 1→1 | 连续实测下跳变 | **最严重** |
| D | M: 2→2 | 预测态内漂移 | 低 |

实测数据：98.9% 的 Type 跳变发生在 Measured=1（C 类），主要矛盾是连续实测下仍跳变，非遮挡问题。

**C-Type Fail 与 Context-ABD Fail**：互斥且完整覆盖所有 fail 航迹。含至少一次 C 类 → C-Type Fail；全部为 A/B/D → Context-ABD Fail。

### 2.4 ExistProb@Jump（辅助列）

记录每次跳变帧的 ExistProb 值，不影响 pass/fail 判定。

- ExistProb 是 Measured 状态的累积衰减量（持续 M=2 逐帧降低，回到 M=1 逐帧恢复）
- 高值（≥90）：跳变发生在正常实测环境，纯逻辑问题
- 低值（<70）：跳变前目标已长时间未被实测更新

### 2.5 Measured 振荡检测

判定依据：Measured 字段在 2 秒滑窗内发生 ≥ 3 次 1↔2 切换。

注意：这是测量状态振荡的**事实标记**，不是"遮挡"的因果判定。Measured=2 单独出现不触发此标记。

## 三、预处理管线

```
原始 CSV
  │
  ├─ Step 0: 时间解析 + 七规则航迹切分
  │
  ├─ Step 1: Track_Age 门控（>= 10）
  │
  ├─ Step 2: 短航迹丢弃（< 20 帧）
  │
  ├─ Step 3: 速度门控（Car/Truck 均速 < 30km/h 剔除）
  │
  ├─ Step 4: 时长过滤（Car/Truck/Uncertain >= 5s, Cyclist/Ped >= 2s）
  │
  ├─ Step 5: Measured 振荡标记
  │
  └─ Step 6: 跳变判定 + 上下文分类 + 距离筛选
```

### 3.1 七规则航迹切分（Step 0）

| 规则 | 触发条件 | 处理 |
|------|----------|------|
| A | ID 首次出现 | 新建段 |
| B | uint8 回绕（age[i-1]>=250 且 age[i]<=5） | 同段继续 |
| C | 非回绕递减（age 突降） | 切分（ID 复用） |
| D | 帧间隔 > 500ms | 切分（时间断裂） |
| E | 文件边界 | 切分 |
| F | 空间跳变（可选，默认关闭） | 切分 |
| G | 段帧数 < 20 | 丢弃 |

切分后 ID 格式：`{radar}_{原始ID}_seg{N}`（如 `flr_12_seg0`）

### 3.2 各步门控参数

| 步骤 | 参数 | 默认值 | 说明 |
|------|------|--------|------|
| Step 1 | min_track_age | 10 | 给雷达充分稳定时间 |
| Step 2 | min_traj_frames | 20 | 20Hz下=1秒，过短无统计意义 |
| Step 3 | threshold_kmh | 30 | 仅影响 Car/Truck，排除静止/低速目标 |
| Step 4 | duration | 1:5s, 2:5s, 3:2s, 4:2s, 7:5s | 按类型差异化 |
| Step 5 | window_s / min_flips | 2.0 / 3 | 振荡检测灵敏度 |
| Step 6 | transition_tolerance | 0 | 零容差 |

## 四、系统架构

### 4.1 三层架构

```
┌─────────────────────────────────────────────────┐
│  UI 层（可选）                                    │
│  app.py — Streamlit Web UI（中文界面）            │
├─────────────────────────────────────────────────┤
│  渲染层 render/                                  │
│  curve_plot.py — Type曲线图（matplotlib Agg）    │
│  excel_export.py — Excel报告（openpyxl嵌图）     │
├─────────────────────────────────────────────────┤
│  算法核心层 core/                                │
│  segmenter.py → preprocess.py → oscillation.py  │
│  → judge.py → pipeline.py（编排）               │
│  models.py（数据结构）                           │
└─────────────────────────────────────────────────┘
```

### 4.2 数据流

```
CSV文件 → load_csv() → DataFrame
  → segment_trajectories() → dict{seg_id: DataFrame}
  → gate_age() → drop_short() → speed_gate() → duration_filter()
  → label_all()（振荡标记）
  → evaluate_all() → list[TrajResult]
  → EvalReport
  → render_all()（曲线PNG）
  → export_report()（Excel bytes）
```

### 4.3 模块职责

| 模块 | 职责 |
|------|------|
| `core/models.py` | TrajResult / FilterLog / EvalReport 数据类 |
| `core/segmenter.py` | 七规则切分，输出 {seg_id: DataFrame} |
| `core/preprocess.py` | Step 1-4 门控过滤 |
| `core/oscillation.py` | Step 5 振荡检测与标记 |
| `core/judge.py` | Step 6 跳变判定 + 上下文 + 距离筛选 |
| `core/pipeline.py` | 管线编排 + CSV加载 + 雷达源解析 |
| `render/curve_plot.py` | 双Y轴曲线（Type阶梯线 + Dx距离线） |
| `render/excel_export.py` | Excel生成（双语表头、嵌图、样式） |
| `main.py` | CLI入口（argparse） |
| `app.py` | Streamlit Web UI |

### 4.4 关键数据结构

```python
@dataclass
class TrajResult:
    traj_id: str              # flr_12_seg0
    radar_src: str            # flr/rlr/fr/rr
    dominant_type: int        # 主类（众数）
    transition_count_all: int # 总跳变次数
    transition_count_in: int  # 有效距离内跳变（判定依据）
    verdict: str              # pass/fail
    contexts: list            # ['C','C','A',...]
    transition_detail: list   # ['1→7','7→1',...]
    exist_prob_at_jump: list  # [100, 52, ...]
    has_msr_oscillation: bool
    msr_osc_ratio: float
    traj_df: DataFrame        # 原始数据引用（供绘图）
```

## 六、输出格式

### 6.1 Excel 报告结构

| Sheet | 内容 |
|-------|------|
| Summary | 按雷达+Type汇总（跳变率、C-Type/ABD-Fail、过滤日志） |
| Type1_Car ~ Type7_Uncertain | 逐航迹明细 + 嵌入曲线图 |
| Field Guide | 字段中英双语说明（三列） |
| Config | 运行参数快照 |

### 6.2 明细 Sheet 列定义

| 列 | 说明 |
|----|------|
| Traj ID 航迹编号 | {radar}_{ID}_seg{N} |
| Radar 雷达来源 | flr/rlr/fr/rr |
| Start/End Time | 起止时间戳 |
| Duration(s) / Frames | 持续时长 / 帧数 |
| Dominant Type 主类 | 众数 Type |
| Transitions / Transitions(In) | 总跳变 / 有效距离内跳变 |
| Context 跳变上下文 | A/B/C/D 拼接 |
| ExistProb@Jump | 跳变帧存在概率 |
| MSR Oscillation / Ratio | 振荡标记 / 占比 |
| Speed(km/h) | 均速 |
| Verdict 判定 | pass（绿）/ fail（红） |
| Type Curve 类型曲线 | 嵌入PNG |

### 6.3 曲线图规格

- 左Y轴：Type（1/2/3/4/7 阶梯线）
- 右Y轴：Dx 纵向距离（蓝色实线）
- 有效距离：水平虚线 + 浅蓝底色区间
- 跳变点（范围内）：红色实心圆 + from→to 标注
- 跳变点（范围外）：橙色空心圆
- 振荡区间：灰色底色

### 6.4 排序规则

先按雷达分组（flr → rlr），组内 fail 排最前（按 transition_count_in 降序），然后 pass。

## 七、配置参数一览

```yaml
segmenter:
  wrap_high: 250          # uint8回绕上阈值
  wrap_low: 5             # uint8回绕下阈值
  gap_threshold_ms: 500   # 帧间隔断裂阈值(ms)
  min_traj_frames: 20     # 最小段帧数
  spatial_split_enabled: false  # 空间跳变切分（默认关闭）

min_track_age: 10         # Track_Age门控

speed_gate:
  enabled: true
  types: [1, 2]           # 仅Car/Truck
  threshold_kmh: 30.0

duration:                 # 最小时长(秒)
  1: 5.0                  # Car
  2: 5.0                  # Truck
  3: 2.0                  # Cyclist
  4: 2.0                  # Pedestrian
  7: 5.0                  # Uncertain

oscillation:
  window_s: 2.0           # 滑窗宽度
  min_flips: 3            # 最小翻转次数

judgment:
  transition_tolerance: 0 # 跳变容差
  eval_range_enabled: true
  eval_range:
    flr: 100.0            # 前雷达(m)
    rlr: 50.0             # 后雷达(m)

output:
  plot_dpi: 100
  plot_size: [900, 350]
  embed_curves: true
```

## 八、设计决策记录

| 决策 | 结论 | 依据 |
|------|------|------|
| ExistProb 做遮挡判断？ | 否 | 是 Measured 的累积衰减量，无独立信息 |
| MotionStatus 纳入算法？ | 否 | 96-97% 恒=1，无区分度 |
| 距离裁剪帧？ | 否 | 产生 40.8% 伪跳变 |
| M2 占比过滤？ | 否 | 不降低 fail 率（82%→81%） |
| 每 Type 200 条硬阈值？ | 否 | Type 3/4/7 天然稀少，不可行 |
| 跳变到 Type=7 豁免？ | 否 | 稳定跟踪中变 Uncertain 本身是异常 |
| transition_tolerance 默认值？ | 0 | Track_Age>=10 后任何跳变均为质量问题 |
