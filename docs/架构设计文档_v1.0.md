# 无真值 Type 输出质量评估 — 架构设计 v1.0

> 日期：2026-07-22
> 定位：独立桌面应用，评估雷达目标 Type 分类输出的自洽性

---

## 一、设计原则

1. **算法与 UI 完全解耦**：核心算法层零 UI 依赖，可命令行/脚本/测试独立调用
2. **管线即函数链**：每个 Step 是纯函数（输入 DataFrame → 输出 DataFrame + 日志），可单独测试
3. **配置驱动**：所有阈值、开关、路径均由 YAML 配置控制，代码中无硬编码魔数
4. **单文件可跑**：最小化依赖，`python main.py --input *.csv --config config.yaml` 即可出报告
5. **渐进式 UI**：先实现 CLI 全流程，再叠加 Dash 桌面界面（UI 是可选层）

## 四、核心数据结构（core/models.py）

```python
from dataclasses import dataclass, field
from typing import Optional
import numpy as np
import pandas as pd


@dataclass
class TrajResult:
    """单条航迹的评估结果"""
    traj_id: str                    # 全局唯一 ID（如 flr_12_seg0）
    radar_src: str                  # 雷达来源（flr/rlr/fr/rr/unk）
    start_time: pd.Timestamp
    end_time: pd.Timestamp
    duration_s: float
    frame_count: int
    dominant_type: int              # 主类（出现帧数最多的 Type）
    mean_speed_kmh: float

    # 跳变判定
    transition_count_all: int       # 总跳变次数
    transition_count_in: int        # 有效距离内跳变次数（判定依据）
    verdict: str                    # 'pass' / 'fail'
    transition_indices: np.ndarray  # 跳变帧索引（有效距离内）
    transition_indices_out: np.ndarray  # 跳变帧索引（有效距离外）

    # 上下文标注
    contexts: list[str]             # ['C', 'A', ...] 每次跳变的 ABCD
    transition_detail: list[str]    # ['1→7', '7→1', ...] from→to
    exist_prob_at_jump: list[int]   # [100, 52, ...] 跳变帧 ExistProb

    # 振荡标记
    has_msr_oscillation: bool
    msr_osc_ratio: float
    msr_osc_segments: list[tuple]   # [(t_start, t_end), ...]

    # 原始数据引用（供绘图）
    traj_df: Optional[pd.DataFrame] = field(default=None, repr=False)


@dataclass
class FilterLog:
    """单步过滤日志"""
    step_name: str                  # 'age_gate' / 'drop_short' / ...
    input_count: int                # 输入航迹数
    output_count: int               # 输出航迹数
    removed_count: int              # 剔除数
    removed_ids: list[str] = field(default_factory=list)


@dataclass
class EvalReport:
    """完整评估报告"""
    results: list[TrajResult]       # 所有航迹结果
    filter_logs: list[FilterLog]    # 各步过滤日志
    config: dict                    # 运行时配置快照
    input_files: list[str]          # 输入文件列表
    radar_sources: list[str]        # 涉及的雷达来源

    @property
    def summary_df(self) -> pd.DataFrame:
        """按 雷达+Type 汇总统计"""
        ...

    @property
    def fail_rate(self) -> float:
        """总体 fail 率"""
        ...
```

## 九、关键设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 管线编排方式 | 显式函数链（非 DAG/插件） | 步骤固定（7步），无需动态编排，简单可调试 |
| 航迹数据载体 | dict[str, DataFrame] | 每步输入输出格式一致，无需自定义容器 |
| 图片存储 | 内存 bytes → 直接嵌入 Excel | 避免临时文件管理，openpyxl 支持 BytesIO |
| 配置管理 | 单 YAML + dataclass 校验 | 避免散落各处的默认值，一处修改全局生效 |
| 雷达来源识别 | 文件名解析 + Dx 兜底 | 无需额外元数据文件，兼容现有命名规范 |
| 有效距离筛选 | 不裁帧，只筛选计入判定的跳变 | 避免伪跳变（40.8%）和碎片化问题 |
| UI 框架 | Dash + pywebview（可选层） | 已有团队经验，跨平台，无需前端构建 |
| 测试策略 | 每步独立单测 + 管线集成测试 | 纯函数设计使单测无需 mock |

## 十、扩展预留

1. **批量对比**：多版本软件结果并排对比（同一场景 v1 vs v2 的 fail 率变化）
2. **增量评估**：新 CSV 追加到已有报告，无需全量重跑
3. **自定义 Type 映射**：config 中定义 Type 值→名称映射，适配不同雷达协议
4. **导出格式扩展**：PDF 报告、HTML 交互式报告（Plotly 替代 matplotlib）
5. **CI 集成**：`python main.py --input *.csv --json-summary` 输出 JSON 供流水线消费
