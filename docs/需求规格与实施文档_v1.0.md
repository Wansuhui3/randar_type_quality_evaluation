---

## 一、项目概述

### 1.1 目标

开发一个独立桌面工具，在无外部真值条件下评估雷达目标 Type 分类输出的自洽性。输入雷达航迹 CSV，输出 Excel 评估报告（含嵌入式 Type 曲线图）。

### 1.2 核心判据

有效评估距离内，航迹 Type 发生跳变 → fail；无跳变 → pass。零容差（1 帧跳变即 fail）。

### 1.3 技术栈

| 层 | 技术 | 用途 |
|----|------|------|
| 核心算法 | Python 3.11+ / pandas 2.x / numpy | 数据处理、管线逻辑 |
| 渲染 | matplotlib 3.7+（Agg）/ openpyxl 3.1+ | 曲线图 + Excel 生成 |
| 配置 | PyYAML | 参数管理 |
| UI | Streamlit | 文件上传、参数调整、报告下载 |
| 测试 | pytest | 单元测试 + 集成测试 |

### 1.4 关键设计决策（已确认）

| 决策 | 结论 | 依据 |
|------|------|------|
| 遮挡筛选 | 不加。M2 筛选不改变 fail 率，且样本量骤降 | 11 文件 539 航迹验证：M2<=10% fail 率 81% vs 全量 82% |
| 距离裁剪 | 不裁帧，只筛选哪些跳变计入判定 | 裁帧产生 40.8% 伪跳变 |
| ExistProb | 仅作为跳变帧辅助标注列，不参与判定 | EP 是 Measured 的累积衰减器，无独立信息 |
| MotionStatus | 不纳入 | 96-97% 恒=1，与遮挡/Type 跳变无关 |
| Type=5 | 不存在，整条航迹若出现则过滤 | 协议中已移除 |
| 转换矩阵 | 删除 | 用户确认作用不大 |
| 200条/Type | 不设硬门槛，按实际样本量报告 | 行车场景 Type 3/4/7 天然稀缺 |
| 嵌图性能 | 默认嵌入，可选关闭 | 瓶颈在 matplotlib（120ms/张），openpyxl 嵌图 <1ms/张 |

## 二、功能需求

### 2.1 输入

- 格式：雷达航迹 CSV（逗号分隔，含表头）
- 必需列：timestamp, ID, Dx, Dy, Vx, Vy, Track_Age, Type, Measured, ExistProb
- 多文件：支持同时输入多个 CSV（前雷达 + 后雷达）
- 雷达识别：从文件名解析（flr/rlr/fr/rr），Dx 符号兜底校验

### 2.2 处理管线（7 步）

| Step | 名称 | 输入 | 输出 | 可配置参数 |
|------|------|------|------|-----------|
| 0 | 时间解析 + 航迹切分 | 原始 CSV | segments_dict | wrap_high, wrap_low, gap_threshold_ms, min_traj_frames |
| 1 | Track_Age 门控 | segments_dict | 过滤后 segments | min_track_age=10 |
| 2 | 短航迹丢弃 | segments_dict | 过滤后 segments | min_traj_frames=20 |
| 3 | 速度门控 | segments_dict | 过滤后 segments | speed_gate_types=[1,2], speed_threshold_kmh=30 |
| 4 | 时长过滤 | segments_dict | 过滤后 segments | duration_thresholds={1:5, 2:5, 3:2, 4:2, 7:5} |
| 5 | Measured 振荡标记 | segments_dict | 标注后 segments | msr_osc_window_s=2.0, msr_osc_min_flips=3 |
| 6 | 跳变判定 | segments_dict | list[TrajResult] | transition_tolerance=0, eval_range={flr:100, rlr:50} |

### 2.3 输出

**Excel 报告（.xlsx）：**

- Sheet 1：汇总统计 Summary（按雷达+Type 分组，含 C-Type Fail / Context-ABD Fail）
- Sheet 2~N：按 Type 分（Type1_Car / Type2_Truck / Type3_Cyclist / Type4_Ped / Type7_Uncertain），内部按雷达分组，fail 排前
- Sheet N+1：字段说明 Field Guide（中英双语）
- Sheet N+2：配置参数 Config（本次运行的参数快照）

**明细 Sheet 列（共 16 列）：**

Traj ID / Radar / Start Time / End Time / Duration(s) / Frames / Dominant Type / Transitions / Transitions(In) / Context / Transition Detail / ExistProb@Jump / MSR Oscillation / MSR Osc Ratio / Speed(km/h) / Verdict / Type Curve

**Type 曲线图（每条航迹）：**

- 左 Y 轴：Type 阶梯线（1/2/3/4/7）
- 右 Y 轴：Dx 距离线 + 有效距离阈值虚线
- 有效距离区间：浅蓝底色
- 跳变标记：红色实心（距离内，计入）/ 橙色空心（距离外，不计入）
- MSR 振荡区间：灰色底色
- 标题：`{traj_id} | dom={type} | dur={s}s | trans={in}/{all} | {verdict}`
- 尺寸：900x350 px，dpi=100

### 2.4 UI 需求（Streamlit）

| 功能 | 组件 | 说明 |
|------|------|------|
| 文件上传 | st.file_uploader(multiple) | 支持多 CSV 拖拽上传 |
| 参数调整 | st.sidebar | 有效距离、Track_Age 门限、速度门限等关键参数 |
| 运行按钮 | st.button | 一键执行全管线 |
| 进度反馈 | st.progress + st.status | 显示当前 Step 和已处理航迹数 |
| 结果预览 | st.dataframe + st.pyplot | 汇总表 + fail 航迹曲线预览 |
| 报告下载 | st.download_button | 下载完整 Excel（含嵌图） |
| 配置导出 | st.download_button | 下载当前参数为 YAML |

## 七、约束与注意事项

1. timestamp 格式：`2026_05_15_09_51_30_980`（下划线分隔，末尾毫秒），解析后 strip 空格
2. 帧率 20Hz（50ms 间隔），时间间隔 > 100ms 视为不连续
3. Dx 前雷达为正，后雷达为负；有效距离用 |Dx| 还是 Dx 取决于雷达方向（前雷达 Dx<=100，后雷达 |Dx|<=50 即 Dx>=-50）
4. matplotlib 必须用 Agg 后端（无 GUI 环境）
5. openpyxl 嵌图用 BytesIO，不落盘临时文件
6. Streamlit 大文件处理时用 st.spinner 避免页面无响应
7. 所有中文输出需兼容 Windows GBK 终端（避免特殊 Unicode 符号）
