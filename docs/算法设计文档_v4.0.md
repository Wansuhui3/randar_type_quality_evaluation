# 无真值 Type 输出质量评估算法 v4.0

> 版本：v4.0
> 日期：2026-07-22
> 定位：在无外部真值（相机/人工标注）条件下，评估雷达目标 Type 分类输出的自洽性与稳定性，用于发现不可靠输出、比较软件版本、筛查待标注样本。

---

## 一、核心判定逻辑

**唯一判据：有效评估距离内，航迹 Type 是否发生跳变。**

- 有效距离内无跳变（transition_count_in == 0）→ **pass**（类型输出一致）
- 有效距离内有跳变（transition_count_in > 0）→ **fail**（类型输出不一致）

有效距离：前雷达（flr/fr）默认 100m，后雷达（rlr/rr）默认 50m，可配置。超出有效距离的跳变记录但不计入判定（曲线图中以橙色空心圆标注）。`eval_range_enabled: false` 时退化为全航迹评估。

无容差：门控后即使仅 1 帧跳变也判 fail。理由：Track_Age >= 10 已给予雷达充分稳定时间，此后任何 Type 变化均为输出质量问题。

**跳变到 Uncertain(7) 同样判 fail**：Type 从任何已知类别（Car/Truck/Cyclist/Pedestrian）跳变为 7（Uncertain），或从 7 跳变为已知类别，均计为一次跳变。Uncertain 不享有任何豁免——目标在稳定跟踪期间突然变为"不确定"，本身就是分类输出异常。

**不输出**：分类准确率、Recall、F1、属性矛盾率、模糊区占比。

**核心指标**：跳变航迹占比（per 雷达+Type）= fail 航迹数 / 该组合总航迹数。

---

## 二、Type 定义

| Type 值 | 含义 | 是否参与评估 |
|---------|------|:---:|
| 1 | Car | 是 |
| 2 | Truck | 是 |
| 3 | Cyclist | 是 |
| 4 | Pedestrian | 是 |
| 7 | Uncertain | 是（跳变到/从 7 均计为跳变） |

## 三、预处理管线

```
原始 CSV
  │
  ├─ Step 0: 时间解析 + 航迹切分（复用已有 segmenter 七规则 A-G）
  │
  ├─ Step 1: Track_Age 门控（>= 10）
  │
  ├─ Step 2: 短航迹丢弃（< min_traj_frames）
  │
  ├─ Step 3: 速度门控（仅 Car/Truck，整条航迹均速 < 30km/h 剔除）
  │
  ├─ Step 4: 时长过滤（Car/Truck/Uncertain >= 5s，Cyclist/Ped >= 2s）
  │
  ├─ Step 5: Measured 状态振荡标记（measurement_state_oscillation）
  │
  └─ Step 6: 跳变判定（有效距离筛选）→ pass / fail
```

每步过滤的剔除数量记入日志，汇总 Sheet 中展示各步过滤比例。

### Step 0：时间解析 + 航迹切分

**时间解析**（显式创建 ts 列，后续所有步骤依赖）：

```python
df['ts'] = pd.to_datetime(df['timestamp'], format='%Y_%m_%d_%H_%M_%S_%f')
df = df.sort_values(['ID', 'ts']).reset_index(drop=True)
```

**航迹切分**：复用已有 `radar_wave_analyzer/core/segmenter.py` 的七规则切分算法（规则 A-G），而非仅依赖 Track_Age 回零。七规则覆盖：

| 规则 | 触发条件 | 动作 |
|------|----------|------|
| A | ID 首现 | 开启新段 |
| B | uint8 回绕（age[i-1]>=250 且 age[i]<=5） | 不切分，同段延续 |
| C | 非回绕下降（diff<0 且不满足 B） | 切分（ID 复用） |
| D | 帧间时间差 > gap_threshold_ms（默认 500ms） | 切分 |
| E | file_index 变化 | 强制切分 |
| F | 帧间速度 > MAX_TRACK_SPEED 或位置跳变 > 阈值 | 切分（可选） |
| G | 生命周期状态跳变（结束集→开始集） | 强制切分（可选） |

```python
from radar_wave_analyzer.core.segmenter import segment_trajectories as seg_existing

meta_df, segments_dict = seg_existing(df, config)
# meta_df: 每段一行（trajectory_id, start_time, end_time, total_frames, is_abnormal...）
# segments_dict: {trajectory_id: DataFrame}
```

**雷达来源前缀**：在 trajectory_id 前拼入雷达标识，解决跨雷达 ID 冲突。

```python
def parse_radar_source(csv_path):
    """从文件名解析雷达位置标识"""
    fname = os.path.basename(csv_path).lower()
    if 'flr' in fname:
        return 'flr'    # 前雷达 Front
    elif 'rlr' in fname:
        return 'rlr'    # 后雷达 Rear
    elif 'fr' in fname:
        return 'fr'     # 前右 Front-Right
    elif 'rr' in fname:
        return 'rr'     # 后右 Rear-Right
    else:
        return 'unk'    # 无法识别，标注警告

# 拼入前缀
radar_src = parse_radar_source(csv_path)
for traj_id in segments_dict:
    new_id = f'{radar_src}_{traj_id}'  # 如 flr_12_seg0
    segments_dict[new_id] = segments_dict.pop(traj_id)
```

输出：每条航迹获得全局唯一 `traj_id`（格式 `{radar_src}_{ID}_seg{N}`，如 `flr_12_seg0`、`rlr_12_seg0`）。

**兜底校验**：若文件名解析失败（radar_src='unk'），可用 Dx 符号辅助判断——前雷达目标 Dx 绝大多数为正（目标在车前），后雷达 97.2% 为负（目标在车后）。若一条"flr"航迹全程 Dx < 0，标注警告：疑似后雷达数据被错误命名。

### Step 1：Track_Age 门控

**问题**：航迹初始帧（Track_Age < 10）Type 跳变率高达 12.2%，属于雷达初始化抖动，非真实分类问题。

**方法**：删除 Track_Age < 10 的帧。

```python
df = df[df['Track_Age'] >= config['min_track_age']]  # 默认 10
```

门控后跳变率降至 < 1%。

### Step 2：短航迹丢弃

**问题**：切分后帧数极少的航迹段（如仅 1-2 帧）无法有效评判 Type 稳定性。

**方法**：丢弃帧数 < min_traj_frames 的航迹段。

```python
min_frames = config['min_traj_frames']  # 默认 20（与已有 segmenter 配置一致）
valid_trajs = [tid for tid, seg in segments_dict.items() if len(seg) >= min_frames]
segments_dict = {tid: segments_dict[tid] for tid in valid_trajs}
```

**注意**：此参数与已有 segmenter 的 `MIN_TRAJ_FRAMES`（默认 20）保持一致。

### Step 3：速度门控（仅 Car/Truck）

**问题**：低速场景（堵车、泊车）下 Car/Truck 的 Type 输出不稳定，且非典型工况，不纳入评估。

**方法**：对主类为 Car(1) 或 Truck(2) 的航迹，计算整条航迹的平均相对速度，< 30 km/h 则整条剔除。Cyclist/Ped/Uncertain 不受此限制。

```python
rel_speed_kmh = np.sqrt(df['Vx']**2 + df['Vy']**2) * 3.6
df['rel_speed_kmh'] = rel_speed_kmh

traj_mean_speed = df.groupby('traj_id')['rel_speed_kmh'].mean()
traj_dom_type = df.groupby('traj_id')['Type'].agg(lambda x: x.mode().iloc[0])

# 仅对 Car/Truck 应用速度门控
speed_gate_trajs = traj_dom_type[traj_dom_type.isin([1, 2])].index
slow_trajs = speed_gate_trajs[traj_mean_speed[speed_gate_trajs] < 30.0]
df = df[~df['traj_id'].isin(slow_trajs)]
```

**注意**：整条航迹过滤，非逐帧过滤。

---

### Step 4：时长过滤

**问题**：存在时间极短的目标（< 2s）帧数过少，无法有效评判 Type 稳定性。

**方法**：按航迹主类应用差异化阈值，保留弱势道路使用者样本。

| 主类 | 最短时长 |
|------|---------|
| Car (1) | 5 s |
| Truck (2) | 5 s |
| Cyclist (3) | 2 s |
| Pedestrian (4) | 2 s |
| Uncertain (7) | 5 s |

```python
def filter_by_duration(df, thresholds):
    traj_duration = df.groupby('traj_id')['ts'].agg(
        lambda x: (x.max() - x.min()).total_seconds()
    )
    traj_dom = df.groupby('traj_id')['Type'].agg(lambda x: x.mode().iloc[0])
    traj_threshold = traj_dom.map(thresholds).fillna(5.0)
    valid = traj_duration[traj_duration >= traj_threshold].index
    return df[df['traj_id'].isin(valid)]
```

**注意**：时长用时间戳差值计算，不用 Track_Age（实测 Track_Age 与实际时长相关性仅 0.16-0.45）。

### Step 5：Measured 状态振荡标记（measurement_state_oscillation）

**定位**：此步骤为**上下文关联标记**，不是因果解释。Measured 的 1↔2 振荡未被真值验证为物理遮挡，不能直接归因为"遮挡"。该标记用于辅助人工核查时理解跳变发生的测量环境，不作为 pass/fail 判据。

**方法**：滑窗检测 Measured 跳变频率，标记振荡区间。

```python
def detect_msr_oscillation(traj_df, window_s=2.0, min_flips=3):
    """
    在 window_s 时间窗内，Measured 发生 >= min_flips 次 1↔2 切换
    → 该时间段标记为 measurement_state_oscillation 区间。

    注意：这是测量状态振荡的事实描述，不是"遮挡"的因果判定。

    返回：
      has_msr_oscillation: bool
      msr_osc_segments: [(t_start, t_end), ...]
      msr_osc_ratio: 振荡标记帧数 / 航迹总帧数
    """
    measured = traj_df['Measured'].values
    ts = traj_df['ts'].values

    flips = np.where(measured[1:] != measured[:-1])[0] + 1

    if len(flips) < min_flips:
        return False, [], 0.0

    osc_mask = np.zeros(len(traj_df), dtype=bool)
    for flip_idx in flips:
        t_center = ts[flip_idx]
        window_start = t_center - np.timedelta64(int(window_s * 1e9), 'ns')
        window_end = t_center + np.timedelta64(int(window_s * 1e9), 'ns')
        in_window = (ts >= window_start) & (ts <= window_end)
        flips_in_window = np.sum(in_window[flips])
        if flips_in_window >= min_flips:
            osc_mask[in_window] = True

    segments = _mask_to_segments(osc_mask, ts)
    ratio = osc_mask.sum() / len(traj_df)
    return len(segments) > 0, segments, ratio
```

**参数**：

| 参数 | 默认值 | 含义 |
|------|--------|------|
| msr_osc_window_s | 2.0 | 滑窗宽度（秒） |
| msr_osc_min_flips | 3 | 窗内最小 Measured 跳变次数 |

**输出标注**（不改变 pass/fail 判定，仅供上下文关联）：

- `has_msr_oscillation`: 该航迹是否存在 Measured 振荡区间
- `msr_osc_segments`: 振荡时间段列表
- `msr_osc_ratio`: 振荡标记帧占比

### Step 6：跳变判定

对每条航迹，按时间排序后统计相邻帧 Type 变化次数，并根据有效距离筛选计入判定的跳变：

```python
def judge_trajectory(traj_df, transition_tolerance=0, eval_range=None):
    """
    eval_range: float or None. 有效距离阈值(m)。
                None = 不启用距离筛选，全部跳变计入判定。
    """
    traj_sorted = traj_df.sort_values('ts')
    types = traj_sorted['Type'].values
    dx = traj_sorted['Dx'].values

    # 检测所有相邻帧跳变
    all_transitions = np.where(types[1:] != types[:-1])[0] + 1  # 跳变帧索引

    # 按有效距离筛选
    if eval_range is not None:
        in_range_mask = dx[all_transitions] <= eval_range
        transitions_in_range = all_transitions[in_range_mask]
        transitions_out_range = all_transitions[~in_range_mask]
    else:
        transitions_in_range = all_transitions
        transitions_out_range = np.array([], dtype=int)

    # 判定仅基于有效距离内跳变
    transition_count_in = len(transitions_in_range)
    transition_count_all = len(all_transitions)
    verdict = 'pass' if transition_count_in <= transition_tolerance else 'fail'

    return transition_count_in, transition_count_all, verdict, transitions_in_range, transitions_out_range
```

**注意**：`transition_tolerance` 参数在此处生效（默认 0 = 无容差）。`eval_range` 为 None 时退化为全航迹评估（向后兼容）。

**跳变到 Uncertain(7) 的判定**：跳变检测不区分目标 Type 值，任何相邻帧 Type 不等即计为一次跳变。因此 Car→Uncertain、Ped→Uncertain、Uncertain→Car 等均为 fail，无需额外规则。

**跳变上下文标注**（仅 fail 航迹）：

对每个跳变帧，根据该帧与前一帧的 Measured 状态分类：

| 代码 | 条件 | 含义 | 严重程度 |
|------|------|------|----------|
| A | Measured 1→2 且 Type 变 | 进入预测态时类型丢失 | 中（信号环境问题） |
| B | Measured 2→1 且 Type 变 | 重新捕获时类型改变 | 中（重捕获不稳定） |
| C | Measured 持续=1 且 Type 变 | 连续实测下类型仍跳 | **高（分类逻辑问题）** |
| D | Measured 持续=2 且 Type 变 | 预测态内类型漂移 | 低（外推阶段不稳定） |

```python
def classify_transition_context(traj_df, transition_indices):
    measured = traj_df['Measured'].values
    exist_prob = traj_df['ExistProb'].values
    contexts = []
    ep_at_jump = []
    for idx in transition_indices:
        m_prev = measured[idx - 1]
        m_curr = measured[idx]
        if m_prev == 1 and m_curr == 2:
            contexts.append('A')
        elif m_prev == 2 and m_curr == 1:
            contexts.append('B')
        elif m_prev == 1 and m_curr == 1:
            contexts.append('C')
        else:
            contexts.append('D')
        ep_at_jump.append(int(exist_prob[idx]))
    return contexts, ep_at_jump
```

| Context | 跳变发生时的 Measured 环境代码。A=进入预测态时跳变；B=重新捕获时跳变；C=连续实测下跳变（最严重）；D=预测态内漂移。多个跳变则拼接显示，如"ACD"。 | Measured-state context at each Type transition. A=entering prediction; B=re-acquisition; C=continuous measurement (most severe); D=prediction drift. Concatenated for multiple transitions. |
| Transition Detail | 每次跳变的类型转换方向，格式"from→to"。如"1→7, 7→1"表示先从Car跳到Uncertain再跳回。用于判断哪个类别在流失。 | Direction of each Type change, format "from→to". E.g. "1→7, 7→1" = Car→Uncertain→Car. Shows which class is leaking. |
| ExistProb@Jump | 每次跳变帧的 ExistProb（存在概率）值，多个跳变用逗号分隔（如"100, 52"）。ExistProb 是 Measured 状态的累积衰减量：持续 Measured=2 时逐帧降低，回到 Measured=1 时逐帧恢复。高值（≥90）说明跳变发生在正常实测环境（纯逻辑问题）；低值（<70）说明跳变前目标已长时间未被实测更新。仅辅助人工核查，不影响 pass/fail 判定。 | ExistProb value at each Type transition frame, comma-separated for multiple jumps (e.g. "100, 52"). ExistProb is a cumulative decay of Measured state: decreases during sustained Measured=2, recovers during Measured=1. High (≥90) = jump during normal measurement (logic issue); Low (<70) = target had prolonged no-measurement period. Auxiliary for manual review only, does NOT affect pass/fail. |
| MSR Oscillation | 是否存在 Measured 状态振荡。判定依据：Measured 字段在 2 秒滑窗内发生 >= 3 次 1↔2 切换。注意：这是测量状态振荡的事实标记，不是"遮挡"的因果判定。Measured=2 单独出现不触发此标记。 | Whether Measured state oscillation is detected. Criteria: Measured flips (1↔2) >= 3 times within 2s window. NOTE: This is a factual measurement-state label, NOT a causal "occlusion" diagnosis. Measured=2 alone does not trigger this. |
| MSR Osc Ratio | 被标记为振荡的帧数占航迹总帧数的比例。0=无振荡，1=全程振荡。 | Fraction of frames in oscillation-marked intervals. 0=none, 1=entire trajectory. |
| Verdict | 最终判定。pass=有效评估距离内 Type 无任何跳变（输出一致）；fail=有效距离内存在至少 1 次 Type 跳变（输出不一致）。有效距离外的跳变记录但不影响判定。跳变到 Uncertain(7) 同样判 fail。 | Final verdict. pass = no Type transition within effective eval range; fail = at least 1 transition within range. Out-of-range transitions are recorded but do not affect verdict. Transitions to/from Uncertain(7) count as fail. |
| Dominant Type | 航迹中出现帧数最多的 Type 值，用于分类统计归属。注意：主类归属会掩盖跳变方向，需结合 Transition Detail 列查看具体跳变路径。 | Most frequent Type value, used for grouping. NOTE: Dominant type hides transition direction; refer to Transition Detail column for specific jump paths. |
| Transitions | 相邻帧 Type 值不同的总次数（含有效距离外的跳变）。例如 Type 序列 [1,1,1,7,1,1] 有 2 次跳变（1→7 和 7→1）。 | Total number of adjacent-frame Type changes (including out-of-range). E.g. [1,1,1,7,1,1] has 2 transitions. |
| Transitions(In) | 有效评估距离内的跳变次数，是 pass/fail 的判定依据。0=pass，>0=fail。与 Transitions 的差值即为有效距离外的跳变数（不计入判定）。 | Number of Type transitions within effective eval range. This is the basis for pass/fail. 0=pass, >0=fail. Difference from Transitions = out-of-range count (not counted). |

### 4.4 Type 曲线图（每条航迹）

- X 轴：时间（秒，从航迹起始归零）
- 左 Y 轴：Type（离散 1/2/3/4/7，阶梯线 step plot）
- 右 Y 轴：Dx 纵向距离（米，实线，颜色 #2196F3）
- 有效距离阈值：右 Y 轴水平虚线（前雷达默认 100m，后雷达默认 50m），标注 "Eval Range"
- 有效距离区间：Dx ≤ 阈值的时间段用浅蓝底色（alpha=0.08）标出
- 跳变点（有效距离内）：红色实心圆点，旁边标注 from→to，计入判定
- 跳变点（有效距离外）：橙色空心圆点，旁边标注 from→to（括号），不计入判定
- Measured 振荡区间：灰色底色（图例标注 "MSR Oscillation"，不写"遮挡"）
- 图标题：`{traj_id} | dom={dominant_type} | dur={duration}s | trans={count_in}/{count_all} | {verdict}`
  - trans 格式：有效距离内跳变数/总跳变数（如 "2/3" 表示 3 次跳变中 2 次在有效距离内）
- 图例：Type step / Dx / Eval Range / Jump(in-range) / Jump(out-range) / MSR Oscillation
- 尺寸：900×350 px，dpi=100

## 六、算法边界与局限

1. **不是准确率**：pass 仅表示"Type 输出自洽无跳变"，不代表分类真实正确。一条全程 Type=1 的航迹可能实际是行人，但本算法无法发现。
2. **稳定错误不可见**：如果雷达对某类目标始终输出错误但一致的 Type，本算法判 pass。发现此类问题需要外部真值。
3. **Measured 振荡非因果**：MSR Oscillation 是测量状态振荡的事实标记，不是"遮挡"的因果判定。振荡可能由遮挡引起，也可能由信号处理、多径、航迹维持机制等引起。持续 Measured=2（不振荡）不会被标记。
4. **速度门控仅针对 Car/Truck**：Cyclist/Ped 天然低速，不适用速度门控。
5. **Uncertain(7) 跳变即 fail**：即使目标确实短暂无法分类（如远距离刚进入检测范围），只要 Track_Age >= 10 后出现 Type→7 跳变，仍判 fail。这是有意为之——稳定跟踪阶段不应出现"突然不确定"。
6. **主类归属的局限**：Dominant Type 用于分组统计，但会掩盖跳变方向。Car→Cyclist→Car 的航迹归入 Car 组，无法从主类统计中看出 Cyclist 在被误判。需结合 Transition Detail 列查看具体跳变路径。
